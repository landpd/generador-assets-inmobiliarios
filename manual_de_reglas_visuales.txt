MANUAL DE DISEÑO, MARCA Y ARQUITECTURA DE CÓDIGO (WHITE-LABEL ENGINE)
====================================================================
Este documento es la fuente ÚNICA de verdad. Está dividido en Brand Tokens (Identidad) y Layout Engine (Estructura de Tailwind). NO INVENTES NADA QUE NO ESTÉ AQUÍ.

--------------------------------------------------------------------
MÓDULO 1: BRAND TOKENS (CONFIGURACIÓN ACTUAL: PULPPO)
--------------------------------------------------------------------
Si se cambia de marca, solo se altera este módulo. Actualmente configurado para "Pulppo" (Proptech B2B/B2C).

1.1 COLORES (7 OFICIALES. PROHIBIDO USAR CUALQUIER OTRO):
- Soft Black: `#212322` (Nunca usar #000 o grises oscuros).
- White: `#FFFFFF`
- Yellow: `#F6BE00` (Acento. Máximo 10% de la pantalla. NUNCA como fondo principal).
- Gray: `#B7B7B7`
- Light-gray: `#F3F3F3`
- Sea Green: `#529999` (Solo feedback positivo o funcional).
- Red: `#A52003` (Solo alertas o negativo).
*Nota en código:* Estos colores se inyectan dinámicamente vía Python mediante el diccionario `paleta` (ej. `bg-[{paleta['fondo']}]`).

1.2 TIPOGRAFÍA (SOLO 2 FUENTES):
- HEADINGS (Títulos H1/H2): `EB Garamond`. 
  *Regla Sagrada:* SIEMPRE en peso Regular (`font-garamond font-normal`). NUNCA Bold, NUNCA itálica. Si se ve débil, se sube el tamaño, no el peso. NUNCA usar Title Case (solo mayúscula inicial).
- BODY / TAGS / CAPTIONS: `Nunito Sans` (`font-nunito`).
  *Regla Sagrada:* Solo 3 pesos permitidos: Light (para captions), Regular (para cuerpo de texto), Bold (EXCLUSIVAMENTE para Tags/Etiquetas en UPPERCASE).

1.3 LOGOTIPO:
- Posición INAMOVIBLE: Esquina superior derecha (`top-[80px] right-[80px]`).
- Prohibido rotar, recolorar o ponerle efectos.

1.4 VOZ Y TONO (PARA LLMs COPYWRITERS):
- Directo, profesional, austero, moderno. NO emocional, NO "startup-juguetona".
- Sujeto + verbo + objeto. Datos duros sobre adjetivos ("+640 MDP", no "increíbles ganancias").
- "Brokers / asesores inmobiliarios" (NO agentes). "La plataforma" (NO la app). Pulppo con doble P.

MÓDULO 2: LAYOUT ENGINE (REGLAS GLOBALES DE TAILWIND Y HTML)
--------------------------------------------------------------------
Reglas de renderizado para `html2image` usando Tailwind CSS.

2.1 PROHIBICIONES VISUALES (ESTRICTO):
- ❌ SIN SOMBRAS. NUNCA. Prohibido usar `shadow-sm`, `shadow-2xl`, `drop-shadow`, `text-shadow`.
- ❌ SIN 3D O ELEVACIÓN. Prohibido glow, bevel, emboss, gradientes mezclados.
- ❌ BODY NUNCA CENTRADO. El texto de cuerpo (`<p>`) va SIEMPRE alineado a la izquierda (`text-left`).
- Todo `<head>` debe incluir el reset: `<style> * { box-shadow: none !important; text-shadow: none !important; } </style>`

2.2 SISTEMA DE RECURSOS DINÁMICOS:
- FOTOS: Usa `imagenes_b64.get('fotos', [])` (Array de fotos de Pexels inyectado por Python).
- OVERLAYS: Toda foto usada como fondo DEBE llevar un overlay para legibilidad: `<div class="absolute inset-0 bg-[{bg_color}]/70 z-10"></div>`.
- SVGS: Usa el diccionario inyectado `{SVGS['nombre']}`. (Ej: logo_pulppo_full, logo_pulppo_isotipo, flecha_larga, bookmark).

2.3 ESTRUCTURA DE COMPONENTES (Para Carruseles Educativos de 3 Actos):
- PORTADA (Slide 0): Layout E (Hero Photo). Foto half-bleed o full-bleed con overlay 70%. Título `text-[130px]` (aprox 56pt-75pt a escala). Etiqueta/Tag en UPPERCASE y `font-bold` encima del título. Logo top-right.
- CONTENIDO (Slides intermedios): Layout B (Title+Body+Foto). Estructura 50/50 vertical asimétrica. Contenido izquierda, foto derecha (o viceversa usando `flex-row`). Separadores de 1pt amarillos. Uso de `formatear_texto_html` para procesar Markdown.
- CTA / CIERRE (Último slide): Layout G (Closing Dark). Fondo sólido (`surface-dark`). Mensaje final + CTA de "Guarda este post" con el icono de Bookmark.

2.4 ESTRUCTURA PARA FICHAS DE PROPIEDADES INMOBILIARIAS:
- Mantener la austeridad. Layout F (Card Grid) o Layout B. 
- Atributos (Cuartos, Baños, m²) alineados a la izquierda, divididos por líneas Neutrales de 1pt (`border-[{paleta['gris']}]`).
- Precio como héroe tipográfico en `font-garamond font-normal`.
